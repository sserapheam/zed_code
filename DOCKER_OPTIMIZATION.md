# Оптимизация Docker для выполнения кода

## 🚀 Реализованные оптимизации

### 1. **Пул долгоживущих контейнеров** ✅
**Проблема:** Создание нового контейнера для каждого запуска занимает ~200-500ms

**Решение:** Пул переиспользуемых контейнеров
- Контейнеры создаются заранее и работают постоянно
- После выполнения кода контейнер возвращается в пул
- Экономия: ~80-90% времени на создание контейнера

**Настройка:**
```bash
# В .env файле
DOCKER_POOL_SIZE=5  # Размер пула (по умолчанию: 5)
```

### 2. **Tmpfs для /tmp (RAM диск)** ✅
**Проблема:** Запись временных файлов на диск медленная

**Решение:** Использование RAM вместо диска
```python
tmpfs={'/tmp': 'size=50m,mode=1777'}
```
- Все временные файлы создаются в RAM
- Скорость: ~10-50x быстрее обычного диска
- Автоматическая очистка при рестарте контейнера

### 3. **Предварительный прогрев контейнеров** ✅
**Проблема:** Первый запуск Python интерпретатора медленный

**Решение:** Автоматический прогрев при создании
```python
warmup_code = "print('warmup')"
```
- Инициализация Python интерпретатора
- Загрузка стандартной библиотеки
- Первое выполнение всегда быстрое

**Настройка:**
```bash
# В .env файле
DOCKER_WARMUP=true  # Включить прогрев (по умолчанию: true)
```

### 4. **Health Check контейнеров** ✅
**Проблема:** Сломанные контейнеры остаются в пуле

**Решение:** Автоматическая проверка здоровья
- Проверка статуса контейнера
- Тестовая команда `echo ok`
- Автоматическое пересоздание сломанных контейнеров

### 5. **Статистика и мониторинг** ✅
**Новая функция:** Отслеживание производительности
- Количество выполнений
- Среднее время выполнения
- Cache hit rate (переиспользование контейнеров)
- Error rate (процент ошибок)

**Доступ к статистике:**
```bash
GET /api/docker/stats
```

**Пример ответа:**
```json

```

## ⚙️ Настройка оптимизаций

### Переменные окружения (.env)

```bash
# Docker режим
USE_DOCKER=true                    # Включить Docker (по умолчанию: false)

# Размер пула
DOCKER_POOL_SIZE=5                 # Количество контейнеров (по умолчанию: 5)
                                   # Рекомендации:
                                   # - CPU < 4 ядер: 3-4 контейнера
                                   # - CPU 4-8 ядер: 5-7 контейнеров
                                   # - CPU > 8 ядер: 8-12 контейнеров

# Прогрев контейнеров
DOCKER_WARMUP=true                 # Включить прогрев (по умолчанию: true)
                                   # false - быстрее создание, медленнее первое выполнение
                                   # true - медленнее создание, быстрее все выполнения

# Автоудаление контейнеров (для отладки)
DOCKER_AUTO_REMOVE=true            # Удалять контейнеры автоматически
```

### Оптимальные настройки для разных случаев:

#### Разработка (мало запросов):
```bash
DOCKER_POOL_SIZE=3
DOCKER_WARMUP=true
```

#### Продакшн (много запросов):
```bash
DOCKER_POOL_SIZE=8
DOCKER_WARMUP=true
```

#### Отладка:
```bash
DOCKER_POOL_SIZE=1
DOCKER_WARMUP=false
DOCKER_AUTO_REMOVE=false  # Контейнеры останутся для проверки
```

## 🎯 Best Practices

### 1. **Размер пула должен соответствовать нагрузке:**
- Слишком мало → создание новых контейнеров во время пиков
- Слишком много → лишнее потребление памяти

### 2. **Мониторить cache_hit_rate:**
- > 95% - отлично ✅
- 80-95% - нормально ⚠️
- < 80% - увеличить пул ❌

### 3. **Следить за error_rate:**
- < 1% - отлично ✅
- 1-5% - проверить логи ⚠️
- > 5% - проблема с кодом/контейнерами ❌

### 4. **Tmpfs размер:**
- Минимум: 50MB (по умолчанию)
- Рекомендуется: 100MB
- Максимум: зависит от доступной RAM

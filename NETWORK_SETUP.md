# Настройка внешнего доступа к Flask приложению

## 🚀 Быстрый старт

1. **Запустите скрипт настройки** (от имени администратора):
   ```bash
   setup_network.bat
   ```

2. **Запустите приложение**:
   ```bash
   .\venv\Scripts\python.exe main_postgres.py
   ```

## 📋 Пошаговая настройка

### 1. Настройка Flask приложения ✅

Приложение уже настроено для внешнего доступа:
- `host="0.0.0.0"` - принимает подключения с любого IP
- `port=5000` - работает на порту 5000

### 2. Настройка брандмауэра Windows

**Вариант A: Автоматически (рекомендуется)**
```bash
# Запустите от имени администратора
setup_network.bat
```

**Вариант B: Вручную**
1. Откройте "Панель управления" → "Система и безопасность" → "Брандмауэр Windows"
2. Нажмите "Дополнительные параметры"
3. Выберите "Правила для входящих подключений" → "Создать правило"
4. Выберите "Порт" → "Далее"
5. Выберите "TCP" → "Определенные локальные порты" → введите `5000`
6. Выберите "Разрешить подключение" → "Далее"
7. Выберите все профили → "Далее"
8. Введите имя: "Flask App Port 5000" → "Готово"

### 3. Получение IP адресов

**Локальный IP (для доступа из локальной сети):**
```bash
ipconfig
```
Ищите "IPv4-адрес" вашей сетевой карты.

**Внешний IP (для доступа из интернета):**
- Перейдите на https://whatismyipaddress.com/
- Скопируйте ваш внешний IP

### 4. Настройка роутера (Port Forwarding)

Для доступа из интернета нужно настроить проброс портов:

1. Откройте веб-интерфейс роутера (обычно http://192.168.1.1 или http://192.168.0.1)
2. Найдите раздел "Port Forwarding" или "Виртуальные серверы"
3. Добавьте правило:
   - **Внешний порт**: 5000
   - **Внутренний порт**: 5000
   - **Внутренний IP**: ваш локальный IP (например, 192.168.1.100)
   - **Протокол**: TCP
   - **Статус**: Включено

## 🌐 Доступ к приложению

### Из локальной сети:
```
http://ВАШ_ЛОКАЛЬНЫЙ_IP:5000
```
Пример: `http://192.168.1.100:5000`

### Из интернета:
```
http://ВАШ_ВНЕШНИЙ_IP:5000
```
Пример: `http://85.123.456.789:5000`

## ⚠️ Важные предупреждения

### Безопасность:
1. **Отключите debug режим** для продакшена:
   ```python
   app.run(host="0.0.0.0", port=5000, debug=False)
   ```

2. **Используйте HTTPS** для защиты данных:
   ```python
   app.run(host="0.0.0.0", port=5000, debug=False, ssl_context='adhoc')
   ```

3. **Настройте аутентификацию** и ограничьте доступ

4. **Используйте переменные окружения** для секретных данных

### Производительность:
- Для продакшена используйте WSGI сервер (Gunicorn, uWSGI)
- Настройте reverse proxy (Nginx)
- Используйте базу данных в продакшене

## 🔧 Дополнительные настройки

### Переменные окружения для продакшена:
```env
FLASK_ENV=production
FLASK_DEBUG=False
DB_HOST=your_production_db_host
DB_PASSWORD=secure_production_password
FLASK_SECRET=very_secure_secret_key
```

### Запуск с Gunicorn (Linux/macOS):
```bash
pip install gunicorn
gunicorn -w 4 -b 0.0.0.0:5000 main_postgres:app
```

### Запуск с Waitress (Windows):
```bash
pip install waitress
waitress-serve --host=0.0.0.0 --port=5000 main_postgres:app
```

## 🐛 Решение проблем

### Проблема: Не могу подключиться извне
**Решение:**
1. Проверьте брандмауэр Windows
2. Проверьте настройки роутера
3. Убедитесь, что приложение запущено с `host="0.0.0.0"`

### Проблема: Ошибка "Connection refused"
**Решение:**
1. Проверьте, что приложение запущено
2. Проверьте правильность IP адреса и порта
3. Проверьте настройки брандмауэра

### Проблема: Медленная работа
**Решение:**
1. Используйте продакшен WSGI сервер
2. Настройте reverse proxy
3. Оптимизируйте базу данных

Если возникли проблемы:
1. Проверьте логи приложения
2. Проверьте настройки сети
3. Убедитесь в правильности всех настроек

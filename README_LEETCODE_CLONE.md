# LeetCode Clone - Руководство пользователя

## Обзор изменений

Приложение было полностью переработано для работы с таблицей `real_leetcode_problems` и приведено к виду, максимально похожему на LeetCode.

## Основные изменения

### 1. База данных
- Переход с SQLite на PostgreSQL
- Использование таблицы `real_leetcode_problems` вместо старых таблиц `tasks` и `categories`
- Новые поля: `difficulty`, `acceptance_rate`, `topic_tags`, `examples`, `constraints`

### 2. Структура приложения
- **Главная страница** (`/`) - показывает популярные темы и недавние задачи
- **Страница задач** (`/problems`) - список всех тем с возможностью фильтрации по сложности
- **Детальная страница задачи** (`/problem/<id>`) - описание задачи с примером кода
- **Поиск** (`/search`) - поиск по названию, сложности и теме
- **Отправки** (`/submissions`) - история решений пользователя

### 3. Новые функции
- **Темы задач** - группировка по `topic_tags` (Array, String, Hash Table, etc.)
- **Уровни сложности** - Easy, Medium, Hard с цветовой индикацией
- **Статистика принятия** - процент успешных решений
- **Теги задач** - отображение связанных тем
- **Примеры и ограничения** - детальное описание задач

### 4. UI/UX улучшения
- LeetCode-стиль цветовая схема для уровней сложности
- Улучшенные карточки задач с тегами
- Адаптивный дизайн для мобильных устройств
- Современный интерфейс с анимациями

## Как использовать

### Для пользователей:
1. **Регистрация/Вход** - создайте аккаунт или войдите в существующий
2. **Просмотр задач** - перейдите в раздел "Задачи" для выбора темы
3. **Фильтрация** - используйте фильтры по сложности (Easy/Medium/Hard)
4. **Решение задач** - нажмите на задачу для просмотра описания и решения
5. **Отправка кода** - напишите решение на Python и отправьте на проверку

### Для разработчиков:
1. **Запуск приложения**:
   ```bash
   python main_postgres.py
   ```

2. **Настройка базы данных**:
   - Убедитесь, что PostgreSQL запущен
   - Проверьте настройки подключения в переменных окружения
   - Таблица `real_leetcode_problems` должна быть заполнена данными

3. **Структура проекта**:
   - `main_postgres.py` - основное приложение
   - `templates/` - HTML шаблоны
   - `static/style.css` - стили CSS
   - `requirements_postgres.txt` - зависимости

## Технические детали

### API Endpoints:
- `GET /` - главная страница
- `GET /problems` - список задач
- `GET /problems?topic=<topic>&difficulty=<level>` - фильтрация задач
- `GET /problem/<id>` - детальная страница задачи
- `POST /problem/<id>` - отправка решения
- `GET /search` - поиск задач
- `GET /submissions` - история отправок
- `GET /api/topics/<topic>/problems` - API для получения задач по теме

### База данных:
- Таблица `real_leetcode_problems` содержит все задачи LeetCode
- Таблица `solutions` хранит решения пользователей
- Таблица `users` - информация о пользователях

## Возможные улучшения

## Заключение

Приложение теперь полностью соответствует концепции LeetCode с современным интерфейсом и функциональностью. Все основные компоненты работают с новой структурой базы данных, а пользовательский интерфейс максимально приближен к оригинальному LeetCode.
